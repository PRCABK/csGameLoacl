# Frontend Development Guidelines

The frontend is a browser-native HTML/CSS/JavaScript application using Three.js. It has no UI framework, hook system, state library, TypeScript, bundler, or automated test suite.

| Guide | Scope |
|---|---|
| [Directory Structure](./directory-structure.md) | Runtime modules, static assets, and separate landing page |
| [UI and Component Guidelines](./component-guidelines.md) | Static DOM, imperative UI, CSS, and controls |
| [Stateful Logic Guidelines](./hook-guidelines.md) | Event/resource lifecycle without framework hooks |
| [State Management](./state-management.md) | Module state, server authority, interpolation, and async tokens |
| [Type Safety](./type-safety.md) | Runtime guards and WebSocket contracts in plain JavaScript |
| [Quality Guidelines](./quality-guidelines.md) | Three.js cleanup, module boundaries, and validation |

These guides document the repository as it currently exists rather than prescribing a framework migration.
